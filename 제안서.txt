ㅇ 개선부분
 1) header - fixed처리 ( 직관적인 ui )
	> 기존 페이지의 header부분은 스크롤을 올려야만 header창이내려왔습니다. 스크롤을 내릴 때는 자동으로 header창이 들어가게되는데, 처음 홈페이지를 방문하였을 때 header창이 어떻게 하면 내려오는지에대한 메커니즘을 몰라 헷갈렸습니다. => 항상 	fixed되게 수정하였습니다. // 사용자들이 좀 더 편하게 사용할 수 있음을 기대할 수 있습니다.

 3) fixed-plusbtn 페이지이동
	> 소비자가 원하는 부분으로 빠르게 이동할 수 있게 빠른 스크롤이동 기능을 넣었습니다. // 사용자들이 좀 더 편하게 사용할 수 있음을 기대할 수 있습니다.

 4) 가로스크롤 속도 향상 ( 상>하 또는 하>상 이동시 너무 답답 )
	> 세로스크롤이 막히고 가로스크롤로 바뀌는 구간이있는데 원래 홈페이지에서의 가로스크롤 속도가 느려 페이지 서칭 시 불편하였습니다. 속도를 빠르게 하여 사용자의 답답함을 해소하였습니다.

 5) 맥북에서 사이트를 열면 스크롤이 빠르게 되는 현상 개선
	> 스크롤이 부드럽게 될 수 있도록 수정하였습니다.

 6) 팝업창이 열릴 시 배경 블러처리 및 스크롤 방지
	> 처음 홈페이지에 들어오면 광고 팝업창이 뜨는데, 뒤에 배경이 스크롤 되니 광고에 집중하기 어렵겠다는 생각이 들었습니다. 그래서 팝업창이 떠 있을 때는 배경의 스크롤 처리를 막았고, 광고의 주목성을 높이기위해 약간의 블러를 추가하여 더욱 			팝업요소에 집중할 수 있게 유도하였습니다.

7) 모바일 화면에서의 광고팝업 레이아웃 깨짐현상 수정
	> 모바일 화면에서의 광고팝업 레이아웃이 약간 깨지는것을 수정하였습니다.
